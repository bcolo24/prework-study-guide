HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • Semantic elements: name of the element describes its content and role.
  • Opening tags appear inside the angle brackets, and closing tags appear inside the angle brackets with a forward slash.
  • The p element represents a paragraph or block of text.
  • The ul, ol, and li elements represent unordered lists, ordered lists, and list items.
  • The img element contains information about images that are displayed on the webpage.
  • The src attribute defines the location of the image.
  • The alt attribute conatins a text string that describes the appearance and functionality of the image.
  • The br element creates a line of empty space, or line break, between two blocks of content.
  • The a element, which stands for anchor, creates links to the same webpage or other webpages.
  • The href attribute points to the URL for the link.

CSS

  • An external CSS style sheet is the preferred method because it allows developers to keep all their CSS rules in a separate file.
  • Declarations contain two important components
    • the CSS property
    • the value of the property
  • Two or more elements that will share the same styling can be grouped together using a comma.
  • Margin: indicates how much space we want around the outside of an element.
  • Padding: indicates how much space we want around the content inside an element.
  • The img selector assigns property values to any image element in the markup
  • A link element creates the connection to the external file.
  • The rel attribute specifies the relationship between the current document and the linked document resource.
  • The href attribute specifies the location (URL) of the external resource.
  • The .card selector contains width, margin, border, and box-shadow properties.
  • The box-shadow CSS property adds shadow effects around an element's frame.
  • Background-color the color behind an element's content and padding.

Git

  • git status: checks what branch we are currently on.
  • git checkout -b branch-name: creates a new branch and switches to it.
  • git add [file]: add a file as it looks now to your next commit (stage).
  • git reset [file]: unstage a file while retaining the changes in working directory.
  • git diff: diff of what is changed but not staged
  • git diff --staged: diff of what is staged but not yet committed
  • git commit -m “[descriptive message]”: commit your staged content as a new commit snapshot.
  • git checkout: switch to another branch and check it out into your working directory
  • git merge [branch]: merge the specified branchs history into the current one
  • git branch: list your branches. a * will appear next to the currently active branch
  • git push [alias] [branch]: Transmit local branch commits to the remote repository branch
  • git pull: fetch and merge any commits from the tracking remote branch

JavaScript

  • Variable: a named container that allows us to store data in our code
  • Strings are contained within quotation marks.
  • Each line of JavaScript code ends with a semicolon.
  • Control flow is the order in which a computer executes code in a script.
    • JavaScript: top to bottom
  • Conditional statements and loops control the order in which code is executed.
  • If statements: code will only execute if the statement meets a condition, or is true.
  • An array is a single variable that is used to hold a group of data.
    • They are usually related in some way.
  • To create an array: assign a variable to a group of values.
  • The first item of an array will be 0, not 1.
  • For loops repeat a pattern until it is no longer true.
  • Three traits of For Loops:
    1. starting point for the loop, ex: x = 0
    2. the condition
    3. what allows the array to iterate over each item, ex: x++, an increase of 1 each time
  • Use the console.log in ChromeDev to see JavaScript run.
